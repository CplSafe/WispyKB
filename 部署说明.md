# WispyKB 服务器部署说明

## 已创建的部署文件

已为你准备了完整的服务器部署文件包，位于 `deployment/` 目录：

```
deployment/
├── README.md                    # 部署说明文档
├── QUICKREF.md                  # 快速参考卡片
├── BAOTA_SETUP.md               # 宝塔面板安装和配置指南
├── backend.Dockerfile           # 后端 Docker 镜像文件
├── docker-compose-prod.yml      # 生产环境配置
├── server-deploy.sh             # 一键部署脚本
└── .env.production.example      # 环境变量模板
```

## 服务器部署步骤

### 步骤 1：上传部署文件

将整个 `deployment/` 目录上传到服务器的 `/data/WispyKB/` 目录：

```bash
# 在本地执行
scp -r deployment/ user@your-server-ip:/data/WispyKB/
```

### 步骤 2：启动模型服务

SSH 登录服务器，启动 DeepSeek 和 Qwen3 模型：

```bash
# SSH 登录服务器
ssh user@your-server-ip

# 确保模型服务已启动
cd /data
./start_models.sh

# 验证模型服务
curl http://localhost:8000/v1/models          # DeepSeek
curl http://localhost:8003/health            # Qwen3
```

### 步骤 3：部署后端服务

```bash
cd /data/WispyKB
chmod +x deployment/server-deploy.sh
./deployment/server-deploy.sh
```

部署脚本会自动：
- ✅ 生成 `requirements.txt`（如果缺失）
- ✅ 创建 `.env` 配置文件
- ✅ 构建 Docker 镜像
- ✅ 启动 PostgreSQL、Redis、后端服务

### 步骤 4：初始化数据库

```bash
# 等待容器启动（约 20 秒）
sleep 20

# 初始化数据库和创建默认管理员
docker exec wispykb-backend python -c "
from core.database import setup_database, create_default_user
import asyncio
asyncio.run(setup_database())
asyncio.run(create_default_user())
"
```

### 步骤 5：安装宝塔面板（管理前端）

参考 `deployment/BAOTA_SETUP.md` 文件进行安装和配置。

## 快速验证

```bash
# 检查所有服务状态
docker-compose -f deployment/docker-compose-prod.yml ps

# 测试后端 API
curl http://localhost:8888/api/v1/health

# 访问 API 文档
# 浏览器打开: http://your-server-ip:8888/docs
```

```bash

docker-compose -f deployment/docker-compose-prod.yml logs -f backend

# 检查模型服务

nvidia-smi

# 数据库备份
docker exec wispykb-postgres pg_dump -U wispykb_user wispykb > backup.sql
```

## 关键点总结

1. **requirements.txt 已自动生成**：部署脚本会自动在后端项目目录创建 `requirements.txt`，无需手动创建。

2. **模型服务已配置**：DeepSeek (8000) 和 Qwen3 (8003) 的启动脚本和 Python 服务都已准备好。

3. **Docker Compose 配置**：使用 `network_mode: host` 模式，后端容器可以直接访问宿主机的模型服务。

4. **宝塔面板管理前端**：推荐使用宝塔面板部署前端，提供可视化的 Nginx 配置和 SSL 证书管理。
